Easy

Reflected XSS

User input is immediately reflected in the server response without proper encoding. The malicious script is not stored but executed immediately.

Immediate response reflection
URL parameter injection
Search/form field vulnerabilities
Error message reflection
Start Challenge
Medium

Stored XSS

User input is stored on the server and displayed to other users without sanitization. This is the most dangerous type as it affects multiple users over time.

Persistent storage in database
Multi-user impact
Comment system vulnerability
Cookie stealing scenarios
Start Challenge
Hard

DOM-Based XSS

Client-side JavaScript processes user input unsafely and writes it to the DOM. The vulnerability exists entirely in client-side code.

Client-side only vulnerabilities
Fragment identifier attacks
Unsafe JavaScript functions
Dynamic script injection
